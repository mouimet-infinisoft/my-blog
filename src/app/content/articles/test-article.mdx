## Test Article with Separate JSON Metadata

This is a test article to verify our new approach of separating metadata from MDX content.

### Benefits of this approach

1. **Clean separation of concerns**: Metadata is stored in JSON files, content in MDX files
2. **Easier to update**: No need to parse MDX files to update metadata
3. **Better type safety**: JSON files can be validated against a schema
4. **Simpler implementation**: No need for complex string manipulation or regex

### Example code

```javascript
// Reading metadata from JSON file
const metadata = JSON.parse(fs.readFileSync('article.json', 'utf8'));

// Using metadata in MDX file
import article from './article.json';
```

This approach makes it much easier to manage content and metadata separately.
